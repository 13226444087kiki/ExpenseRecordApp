项目名称
ExpenseRecordApp
项目简介
这是一个基于Android平台的个人消费和收入记录管理应用。应用允许用户记录日常收支情况，支持数据的持久化存储，帮助用户更好地管理个人财务。
主要功能
支持添加收入和支出两种类型的记录，用户可以填写金额、选择类别、设置日期时间并添加备注。首页展示费用记录列表，支持左滑删除操作。提供详细记录页面查看完整的记账记录。界面上收入和支出用不同颜色区分，便于识别。应用还包含ContentProvider测试、广播接收器、后台服务和通知功能的演示模块。
使用技术
采用Java语言开发，基于Android SDK API 24及以上版本。使用ViewModel管理数据，RecyclerView展示列表数据。数据通过SQLite数据库持久化存储，通过ContentProvider提供数据访问接口。UI方面使用DatePickerDialog和TimePickerDialog实现日期时间选择，ItemTouchHelper实现左滑删除功能。此外还集成了BroadcastReceiver广播接收器、Service后台服务和NotificationManager通知管理器。
项目结构
代码组织在app/src/main/java/com/example/expenserecordapp目录下，包含adapter适配器目录、broadcast广播目录、helper工具类目录、model数据模型目录、service服务目录和ui界面目录。主要文件包括MainActivity主活动、RecordListActivity记录列表活动、ExpenseViewModel视图模型、ExpenseProvider内容提供者、ExpenseDatabaseHelper数据库帮助类、ExpenseContract协议定义、Transaction交易模型等。资源文件存放在res目录下，包含layout布局文件、drawable图片资源和values配置文件。
运行方式
需要安装Android Studio Hedgehog或更高版本，配置JDK 17或更高版本。可以通过命令行运行./gradlew assembleDebug构建项目，也可以直接在Android Studio中点击运行按钮。构建完成后将APK安装到Android设备或模拟器上即可运行。
开发过程
首先进行项目初始化，创建Android项目并配置基本UI布局。然后设计SQLite数据库结构，实现ContentProvider和数据库帮助类，定义Transaction和Record数据模型。接下来创建ExpenseViewModel管理数据，实现添加、删除、查询等业务逻辑，集成RecyclerView展示数据。之后添加类型选择、日期时间选择器和左滑删除功能，进行界面美化。最后集成广播接收器、后台服务和通知功能等高级特性，并修复开发过程中遇到的变量重复定义、ColumnIndex安全检查等问题。
